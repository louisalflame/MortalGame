# AI 工作大綱 - MortalGame 詳細工作指南

> 💡 **專案背景與技術架構請參見**：[copilot-instructions.md](../.github/copilot-instructions.md)

## 工作原則
1. **優先閱讀**：每次開始任何工作前，必須先閱讀此大綱檔案
2. **系統性分析**：按照系統模組逐一分析，不遺漏任何重要組件
3. **文檔完整性**：確保每個系統都有對應的詳細筆記
4. **交叉引用**：當筆記中提到其他系統時，必須建立適當的引用連結

## 系統模組分析重點

### 核心系統模組
**GameModel** - 核心邏輯層
- 狀態管理、流程控制、事件系統
- 實體系統、動作系統、條件系統、目標系統

**GameView** - 視覺呈現層
- 視覺效果與動畫系統

**UI** - 使用者互動層
- 介面邏輯與用戶交互

**GameData** - 資料定義層
- 所有遊戲資料結構定義

## AI 工作流程

### 文檔分析階段
1. **核心系統優先**：GameModel → GameData → UI → GameView  
2. **記錄系統間相依性**：標記交叉引用關係
3. **更新追蹤系統**：維護 `AI_Notes_Index.md`

## 筆記命名規範

**基本原則**：
- 單一腳本：`CardData.md`
- 系統整合：`Card_System.md`  
- 枚舉參考：`GameEnum_Reference.md`

**命名決策**：
- 單一職責 → 直接使用腳本名
- 系統整合 → 使用 `_System` 後綴
- 避免冗余後綴（如 `_Class`）

## 文檔撰寫標準

> 📝 **詳細文檔撰寫指導原則請參見**：[Documentation_Guidelines.md](Documentation_Guidelines.md)
